# 네트워크란 — DevOps/SRE 관점의 한눈에 잡히는 가이드

> "VPC·서브넷·IP·포트·NAT… 매번 봐도 새롭다"는 분들을 위해. **단 하나의 비유 — 아파트 단지** — 로 모든 개념을 꿰뚫습니다. 챕터·캡스톤 어디서 어떻게 쓰이는지까지.

---

## 🏢 한 문장 큰 그림

> **네트워크 = 아파트 단지.** 컴퓨터(=호실)들이 모여 살고, 단지 안끼리는 자유롭게 왕래하지만, 단지 밖과는 정문이나 경비실 우편함을 거쳐야 합니다.

이 비유 하나로 거의 모든 게 설명됩니다. 외워둘 게 있다면 **이 표 한 장**:

| 네트워크 용어 | 아파트 단지 비유 |
|---|---|
| **IP 주소** | 호실 주소 (101동 1505호) |
| **포트** | 그 호실 안의 *방* (안방=DB, 거실=웹) |
| **사설 IP** | 단지 내부 동·호수 (단지 밖 우편물엔 못 씀) |
| **공인 IP** | 전국 우편 시스템이 알아보는 주소 (도로명+건물번호) |
| **DNS** | 전화번호부 ("쿠팡" → "203.x.x.x") |
| **서브넷** | 단지 안의 한 동(棟) |
| **VPC** | 아파트 단지 전체 |
| **IGW (Internet Gateway)** | 단지 정문 (양방향 통행) |
| **NAT Gateway** | 정문 경비실의 우편 대행 서비스 (내→외만) |
| **라우팅 테이블** | 우편 분류표 ("이 우편물 → 저쪽 트럭으로") |
| **Security Group / 방화벽** | 호실 문지기 ("이 IP는 22번방만 허용") |
| **로드밸런서** | 음식점 큐 매니저 ("이쪽 빈자리로!") |
| **K8s Pod IP** | 임시 책상번호 (사람 바뀌면 번호도 바뀜) |
| **K8s Service** | 부서 대표번호 (담당자 바뀌어도 같은 번호) |
| **K8s Ingress** | 단지 정문 안내데스크 ("쿠팡 방문 1층, 카카오 2층") |
| **NetworkPolicy** | 동별 출입 화이트리스트 ("9동 사람은 7동만") |

→ 다 이해 안 돼도 OK. **이 표를 책갈피처럼 두고 필요할 때 돌아오면 됩니다.**

## 🎯 가장 자주 헷갈리는 5가지

### 1. 사설 IP vs 공인 IP — *왜* 분리되어 있나
- **사설**(`10.x.x.x`, `172.16~31.x.x`, `192.168.x.x`) = **단지 내부 주소**. 전 세계 어느 단지든 똑같이 씀. 외부에 보내봐야 의미 없음.
- **공인** = **전국 우편 시스템이 인식하는 진짜 주소**. 한정된 자원이라 비용·할당 받음.
- **왜 분리?** 모든 컴퓨터에 공인 IP 주면 IPv4 주소가 부족함 (총 43억 개뿐 — 이미 고갈). → 단지마다 사설을 쓰고, 단지 정문에서 NAT로 공인 IP 하나를 공유.

### 2. 포트는 IP 다음 단계
- IP = "건물 도착", 포트 = "건물 안 어느 방?"
- `https://쿠팡.com:443` → "쿠팡 건물의 443호(웹 + 보안)로 가세요"
- **잘 쓰이는 포트** (외워두면 절반 이해됨):
  - **22**: SSH (원격 접속)
  - **80**: HTTP (웹, 평문)
  - **443**: HTTPS (웹, 암호화)
  - **53**: DNS
  - **3306**: MySQL / **5432**: PostgreSQL / **6379**: Redis

### 3. NAT는 *내부→외부 한 방향 마술*
- 단지 안 사람이 외부에 우편 보낼 때, **발신자 주소를 단지 정문 주소로 바꿔서** 보냄.
- 답장이 정문에 도착 → 정문 직원이 "아 이거 1505호 우편이었지" 기억하고 1505호로 전달.
- → 외부에서 **먼저 단지 안 1505호로 우편 못 보냄** (정문은 누구한테 갈지 모름).
- 그래서 외부 접속이 필요한 서버(웹)는 공인 IP가 따로 있거나, *Load Balancer* 뒤에 있음.

### 4. 같은 IP인데 *어디서 통하느냐* 차이
- 사설 IP `10.0.1.5`는 **같은 VPC 안에서만** 통함. 다른 VPC의 `10.0.1.5`와 충돌 X.
- 공인 IP `52.79.60.38`은 **전 세계 어디서나** 같은 컴퓨터를 가리킴.
- → "이 IP가 어디서 통하는가"를 항상 생각.

### 5. 라우팅 = "기본은 이쪽으로, 예외는 저쪽으로"
- 라우트 테이블 = "이 목적지로 가려면 어느 게이트웨이 거치라"는 표.
- 가장 흔한 룰:
  - `10.0.0.0/16` → `local` (같은 VPC 안)
  - `0.0.0.0/0` → `IGW` or `NAT GW` (그 외 모든 곳 = 인터넷)
- "퍼블릭 서브넷"이 사실 뭔가? = **라우트 테이블에 `0.0.0.0/0 → IGW`가 있는** 서브넷.

## 🗺 우리 프로젝트의 네트워크 구조 (한눈에)

### 🏠 홈랩 (Ch 01-09 학습 환경)
```
[ Windows 호스트 ]
        │
        │  VMware NAT 192.168.223.0/24
        │
   ┌────┴────┐  ┌────────────┐  ┌────────────┐  ┌────────────┐
   │ master  │  │ worker1    │  │ worker2    │  │ worker3    │
   │ .10     │  │ .11        │  │ .12        │  │ .13        │
   └────┬────┘  └─────┬──────┘  └─────┬──────┘  └─────┬──────┘
        │             │               │               │
        └─── K8s Pod CIDR 10.244.0.0/16 (Calico VXLAN) ────┘
              Service CIDR 10.96.0.0/12

외부 접속:
  nginx.192.168.223.11.nip.io (Ch 04 ingress-nginx + cert-manager)
  grafana.192.168.223.11.nip.io / argocd... / webapp...
```

→ **"단지" 비유**: VMware NAT 네트워크가 하나의 단지(192.168.223.0/24), 그 안에 4개 동(VM), 각 동 안에 K8s가 Pod라는 임시 책상들을 또 깖.

### ☁️ AWS 캡스톤 (Ch 10)
```
[ 인터넷 ]
      │
      ▼
[ ALB (퍼블릭 서브넷에) ]                       ← Phase D
      │
      ▼
┌─────────────────────────────────────────────┐
│  VPC 10.0.0.0/16  (단지 통째)                │
│                                             │
│  ┌────────────────┐    ┌────────────────┐  │
│  │ public-a       │    │ public-c       │  │
│  │ 10.0.1.0/24    │    │ 10.0.2.0/24    │  │ ← ALB·NAT GW
│  │ (AZ-a)         │    │ (AZ-c)         │  │
│  └────────┬───────┘    └────────────────┘  │
│           │ NAT GW (52.79.60.38)            │
│           ▼                                 │
│  ┌────────────────┐    ┌────────────────┐  │
│  │ private-a      │    │ private-c      │  │
│  │ 10.0.11.0/24   │    │ 10.0.12.0/24   │  │ ← EKS 노드·RDS·Redis
│  │ (AZ-a)         │    │ (AZ-c)         │  │
│  └────────────────┘    └────────────────┘  │
│                                             │
│  IGW ─────────────────── 단지 정문         │
└─────────────────────────────────────────────┘
```

→ **"단지" 비유**: VPC 10.0.0.0/16 = 단지. 동(서브넷) 4개. 1·2동(public)은 정문(IGW)에 가까워서 외부 손님 받음. 11·12동(private)은 단지 안쪽이라 *외부 손님 직접 못 받음*. 외부에 우편 보낼 땐 경비실 우편함(NAT GW) 거쳐서. 단지를 두 구역(AZ a, c)으로 나눠서 한쪽 정전이 나도 다른 쪽이 살아있게.

## 💡 학습 팁

1. **"왜?"를 항상 묻기** — "이 NAT 왜 있지?" → "프라이빗 동에서 외부에 우편 보내려고" → 의미가 박힘.
2. **그림으로 그려보기** — 패킷이 어떤 경로로 가는지 손으로 화살표 그리면 외워짐.
3. **`tcpdump`/`traceroute`로 *실제* 보기** — "정말 이 경로로 가는지" 한 번이라도 확인하면 평생 안 잊음.
4. **CIDR 계산기는 그냥 도구로** — 처음엔 `/24`=256, `/16`=65536만 외워도 충분. 나머진 https://cidr.xyz 같은 사이트.
5. **헷갈리면 *비유로 환원*** — "이게 호실 주소인가, 호실 안 방 번호인가?" — IP인지 포트인지 명확해짐.
